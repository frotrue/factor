# Neural Factory Concept

## 한 줄 설명

AI 데이터 처리 공장을 만들고, 침입 포트에서 들어오는 적을 방어하는 공장 자동화/타워 디펜스 게임.

## 플레이어 판타지

플레이어는 Neural Core를 지키는 시스템 운영자입니다. 자원과 데이터를 모아 처리 라인을 만들고, 학습 결과를 방어 모델에 공급해 점점 더 강한 침입을 막습니다.

## 핵심 루프

1. 맵에서 Silicon / Energy 패치를 확인한다.
2. Extractor와 Data Downloader로 자원과 데이터를 얻는다.
3. Processor와 Weight Trainer로 데이터를 가공한다.
4. Core에 Weight Update를 전달해 Confidence Score를 얻는다.
5. 연구와 방어 모델을 강화한다.
6. North/East/South/West Port에서 들어오는 적을 방어한다.
7. 웨이브 압박에 맞춰 공장, 전력망, 방어선을 확장한다.

## 현재 구현된 방향

### 초반 안정감

Normal 기준 Wave 1~10은 North Port 중심으로 진행됩니다. 플레이어는 한 방향을 먼저 방어하면서 게임 규칙을 익힙니다. Wave 11부터 East Port가 추가되어 분산 방어 판단이 시작됩니다.

### 읽히는 방어선

웨이브 예고는 다음 포트, 위협도, 특수 위협, 추천 방어 힌트를 HUD에 보여줍니다. 웨이브 시작 시 포트 라벨과 코어 방향 경로가 표시됩니다.

### 지형과 건물 피해

BLOCKER 지형은 건설과 적 이동을 막아 초반 방어 협곡을 만듭니다. 모든 건물은 HP를 가지며, 적은 코어뿐 아니라 현재/인접 건물을 공격할 수 있습니다. 건물이 파괴되면 연결된 케이블도 정리됩니다.

### 공장과 방어의 결합

방어는 독립 시스템이 아니라 생산 라인과 연결됩니다. Confidence Score와 Model Training Lab은 방어 모델 성능을 올리고, 전력/물류/데이터 병목은 방어 능력에 직접 영향을 줍니다.

## 주요 시스템

- `MapManager`: 자원 패치와 BLOCKER 지형 생성/조회
- `WaveManager`: 웨이브 계획, 포트 선택, 적 스폰, 보상
- `BaseEnemy`: 이동, 경로 탐색, 건물 공격, 특수 상태
- `BaseBuilding`: 공통 HP, 피해, 파괴, 감염 상태
- `CableManager`: 데이터 전송, Fiber/AP, 파괴 시 연결 정리
- `PowerManager`: 전력망 계산, 블랙아웃
- `ResearchManager`: 연구 해금과 효과
- `TutorialManager`: 첫 루프 안내

## 디자인 원칙

- 초반에는 한 방향을 막게 해서 학습 부담을 낮춘다.
- 중반부터 포트와 특수 적을 늘려 판단을 만든다.
- 건물 피해는 읽히는 피드백을 주되, 초반 공장을 과도하게 처벌하지 않는다.
- 지형은 복잡한 규칙보다 방어선 형성을 돕는 장치로 쓴다.
