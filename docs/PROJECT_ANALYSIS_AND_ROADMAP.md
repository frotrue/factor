## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | The Neural Factory |
| **장르** | 공장 자동화(Factorio-like) + 타워 디펜스 + 인크리멘탈 |
| **엔진** | Phaser 3 (v3.90+) |
| **언어** | TypeScript 6.x |
| **빌드** | Vite 5.x |
| **테마** | 사이버펑크 / AI·신경망 |

### 핵심 게임 루프
```
원시 데이터 채굴 → 라벨링 가공 → 가중치 학습 → 모델 훈련 → 추론 유닛 생산
                                                           ↓
                                                    방어 타워에 공급
                                                           ↓
                                                  적(Adversarial Data) 방어
```

## 3. 아키텍처 분석

### 디렉토리 구조
```
src/
├── main.ts                    # Phaser 게임 초기화 (진입점)
├── config.ts                  # 전역 게임 설정 (건물/레시피/적/연구 등)
├── types.ts                   # TypeScript 인터페이스 정의
├── buildings/                 # 건물 클래스
│   ├── BaseBuilding.ts        # 기반 클래스
│   ├── BuildingFactory.ts     # 팩토리 패턴
│   ├── AbstractProcessor.ts   # 가공 건물 공통 로직
│   ├── [12개 건물 클래스]
│   └── legacy/                # 폐기된 컨베이어 관련 클래스
├── enemies/
│   └── BaseEnemy.ts           # 적 클래스 (4종 공유)
├── managers/                  # 게임 시스템 매니저
│   ├── BuildingManager.ts     # 건물 배치/제거
│   ├── CableManager.ts       # 케이블 + AP 데이터 전송
│   ├── CameraController.ts   # 카메라 이동/줌
│   ├── EventBus.ts            # 이벤트 시스템
│   ├── GridRenderer.ts        # 그리드 렌더링
│   ├── InventoryManager.ts    # 자원 인벤토리
│   ├── ItemManager.ts         # 아이템 스프라이트 관리
│   ├── MapManager.ts          # 자원 매립지 생성
│   ├── PowerManager.ts        # 전력망 관리
│   ├── ResearchManager.ts     # 연구/해금
│   ├── SaveManager.ts         # 저장/불러오기
│   ├── TickSystem.ts          # 게임 틱 루프
│   ├── UIManager.ts           # HTML DOM UI 관리
│   └── WaveManager.ts         # 웨이브 스폰/관리
└── scenes/
    ├── MainMenuScene.ts       # 메인 메뉴
    └── MainScene.ts           # 게임 메인 씬
```

### 설계 패턴
- **매니저 패턴**: 각 시스템이 독립된 Manager 클래스로 분리됨
- **팩토리 패턴**: `BuildingFactory`로 건물 인스턴스 생성
- **이벤트 버스**: `EventBus`를 통한 시스템 간 느슨한 결합
- **틱 기반 업데이트**: `TickSystem`이 고정 간격으로 게임 로직 실행

### ⚔️ Phase 1: 게임플레이 핵심 강화 (1~2주)
> 게임의 재미와 깊이를 결정하는 핵심 시스템 개선

#### 1-1. 적 AI 개선
- [ ] A* 또는 BFS 기반 **경로 탐색** 구현 (건물 우회)
- [ ] 적 유형별 **특수 행동** 추가
  - Malware: 건물에 감염 효과 (생산속도 저하)
  - Adversarial: 투명화 (방어 타워 탐지 범위 감소)
  - Overfitted Model: 주변 적에게 버프
- [ ] 적 사망 시 **자원 드랍** 시스템 (소량의 Silicon 획득)

#### 1-2. 밸런스 조정
- [ ] 웨이브별 난이도 곡선 재설계 (로그 스케일링)
- [ ] 방어 타워 **업그레이드 시스템** 도입
  - 같은 타워에 자원 투입 → 사거리/데미지/발사속도 단계별 강화
- [ ] 초반 진입 난이도 완화 (첫 3웨이브 간격 확대)
- [ ] 자원 매립지 크기/분포 밸런스 조정

#### 1-3. 연구 트리 확장
- [ ] 현재 5개 → **15개 이상**으로 확장
- [ ] 분기형 트리 구조 도입 (공격 특화 vs 생산 특화)
- [ ] 추가 연구 예시:
  - `효율적 채굴`: 채굴기 생산량 +50%
  - `대역폭 확장`: 케이블 대역폭 +3

### 🔧 Phase 3: 시스템 확장 (3~4주)
> 장기 플레이를 위한 콘텐츠 추가

#### 3-1. 신규 건물 추가
- [ ] **데이터 복제기 (Data Replicator)**: 입력 데이터를 2개로 복제 (전력 소비 높음)
- [ ] **로드 밸런서 (Load Balancer)**: 연결된 여러 건물에 데이터 균등 분배
- [ ] **캐시 메모리 (Cache)**: 소형 1x1 즉시 저장소 (버퍼 20)
- [ ] **오버클럭 모듈 (Overclock)**: 인접 건물의 처리 속도 2배 (전력 소비 3배)

#### 3-2. 난이도 선택 & 게임 모드
- [ ] 메인 메뉴에 **난이도 선택** 추가 (Easy / Normal / Hard)

- [ ] **무한 모드**: 웨이브 끝없이 진행, 최고 웨이브 기록
- [ ] **챌린지 모드**: 특수 조건 (전력 없이 10웨이브 생존 등)

#### 3-3. 통계 및 업적 시스템
