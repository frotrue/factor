# Gradium Docs

Current docs stay small and operational. Historical plans live in `archive/` when they are still useful for traceability.

## Current Docs

- `PROJECT_MAP.md`: project structure and major systems.
- `ARCHITECTURE.md`: current runtime and UI architecture.
- `FILE_ROLE_MAP.md`: concise ownership map for important files.
- `TEST_MAP.md`: test selection guide by change type.
- `GAME_BALANCE_MAP.md`: current balance values and tuning notes.
- `NEXT_TASKS.md`: active follow-up work.

## Archive

Use `archive/` only for historical context. Current implementation decisions should be reflected in the files above, not in completed plan documents.
